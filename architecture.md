Для масштабирования напрашивается асинхронность. Вот что предлагает AI:
1. Стек технологий
Язык: Python (FastAPI для API, Loguru для логов).
Очереди задач: Celery + Redis. Это позволит разнести запросы во времени и выполнять их параллельно.
Браузерная автоматизация: Playwright (внутри Camoufox) в режиме Connect. Вместо запуска нового браузера подключаемся к пулу уже запущенных инстансов.
База данных: PostgreSQL (хранение истории позиций) + Grafana (для визуализации трендов).
Уведомления: aiogram для асинхронной отправки алертов в Telegram.
2. Устойчивость и обход блокировок
Прокси: Использование резидентских прокси (например, Bright Data или Smartproxy) с ротацией IP на каждый запрос. Это имитирует реальных пользователей из разных регионов.
Rate Limiting: Ограничение до 5–10 одновременных сессий, чтобы не вызывать подозрений у антифрод-систем маркетплейса.
Fingerprinting: Camoufox уже подменяет отпечатки, но важно также рандомизировать User-Agent и задержки между действиями (humanize=True).
Геолокация: Фиксация точки выдачи (склада), так как позиции товара зависят от региона.
3. Архитектура системы
Scheduler (Celery Beat): Каждые N часов генерирует 10 000 задач и кладет их в Redis.
Worker Pool: 5–10 воркеров разбирают задачи. Каждый вокер берет прокси из пула и делает запрос.
Data Layer: Результат пишется в Postgres. Если позиция > 50, срабатывает триггер на отправку сообщения в Telegram-бот.
Monitoring: Flower для контроля очередей и Prometheus для отслеживания % успешных ответов.
4. Риски и их снижение
Капча: При обнаружении капчи воркер должен помечать прокси как «битый» и перезапускать задачу с новым IP. Можно интегрировать сервис решения капчи (2Captcha).
Изменение верстки: Ozon часто меняет селекторы. Решение: использовать тесты-чекеры, которые раз в час проверяют корректность парсинга на эталонных запросах.
Блокировка аккаунта/IP: Использовать только "чистые" сессии без авторизации, если это возможно.
5. Примерный бюджет (в месяц)
